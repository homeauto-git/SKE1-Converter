Binary Format
offset size field value / notes ────────────────────────────────────────────────────────────── 0x00–0x03 4 B File Size ← big-endian uint32, total including header 0x04–0x07 4 B Magic "SKE1" (53 4B 45 31) 0x08–0x0F 8 B Header Block 5B 7C FF FF FF FF FF FF ← confirmed static 0x10–end N B Payload ← raw G.726LE audio ────────────────────────────────────────────────────────────── codec parameters ────────────────────────────────────────────────────────────── codec G.726 ADPCM, little-endian bit order (g726le) sample rate 8000 Hz bitrate 32 kbps (4 bits/sample) channels 1 (mono) container raw / headerless ────────────────────────────────────────────────────────────── ffmpeg cli (manual) ────────────────────────────────────────────────────────────── # Extract payload dd if=in.ske1 of=payload.g726 bs=1 skip=16 # Decode → WAV ffmpeg -f g726le -ar 8000 -i payload.g726 out.wav # Encode WAV → payload ffmpeg -i in.wav -ac 1 -ar 8000 -c:a g726le -b:a 32k -f g726le payload.g726
Notes
The 8-byte block 5B 7C FF FF FF FF FF FF is static across verified samples — it carries no checksum or metadata. The file-size field at 0x00 must be an exact big-endian uint32 of the complete file length or the device will reject it.

G.726LE differs from standard G.726 only in bit ordering within each byte. Using g726 instead of g726le produces garbled audio. The two are not interchangeable.
